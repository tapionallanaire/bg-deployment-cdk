Static Frontend

CloudFront in front, S3 behind it.

This frontend is intentionally simple. It exists to prove that the static delivery path is separate from the backend blue/green flow and can be deployed independently through CDK.

What this validates

The S3 bucket is private, CloudFront reads through Origin Access Control, and viewers are redirected to HTTPS.

What it does not do

The frontend does not implement blue/green routing. The exercise only requires that deployment pattern for the backend API.

Why it is separate

A static asset update should not need to touch ECS services, ALBs, or backend traffic-shift settings.
